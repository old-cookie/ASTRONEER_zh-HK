The parser that comes with Slate though uses a syntax very similar to xml.<lf><lf>          &lt;TextBlockStyleName>Your text content&lt;/><lf>          &lt;img src="SlateBrushStyleName"/><lf>          &lt;a id="YourCustomId"/>Your hyperlink text&lt;/><lf><lf>If you ever want to use the markup syntax as actual text you can escape the markup using xml style escapes. For example:<lf><lf>          <    &amp;lt;<lf>          >    &amp;gt;<lf>          "     &amp;quot;<lf><lf>But you only need to escape these characters when a set of them match actual syntax so this isn't generally an issue.<lf><lf><RichText.Header>Are there any catches?</><lf><lf>There are still plenty of things the SRichTextBlock doesn't currently support. The most notable lacking feature is not having the ability to flow text around images or widgets.
